Provides an interface to the client-remote communication channel. With no
options, displays the current status of the communication channel.
The option '--resync' clears the channel's synchronization state forcing
synchronization to be reestablished automatically on the next command. This
is useful in the case that the remote has been restarted but this client has
not, or to restore synchronization in the case that another client has
connected to the remote and "stolen" synchronization.
The option '--disconnect' will disconnect the current communication channel.
The option '--connect' will create a new communication channel to a remote at
the given address and optional port. Note: the current communication
channel must be disconnected before a new connection can be established.

Usage: getlog [ ... options ... ] [ <entry number> ]
Valid options:
-b [ --begin ] <entry number> : First log entry to display
-e [ --end ] <entry number> : Last log entry to display
-o [ --logfile ] <log file> : Redirect output to <log file>
-d [ --showdata ] : Display log data as human readable
-x [ --showhex ] : Display log data as hexadecimal
Displays the contents of the remote log. If '--begin' and '--end' are
specified output all log entries in that range. If only '--begin' is specified
(or if a non-option <entry_number> is given) output log entries from that index
to the end of the log. If only '--end' is specified output log entries from
the beginning of the log to that index.
The option '--logfile' allows specification of the file name to redirect the
log contents to rather than the standard output. '--showdata' or '--showhex'
will output a portion of the packet that triggered the rule in an ascii or
hexadecimal format respectively.
